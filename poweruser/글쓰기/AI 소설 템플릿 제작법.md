# AI 소설 템플릿 제작법

## Prompt:
생성 AI를 사용하여 템플릿화 된 소설을 만들고자 한다. 
가장 효과적인 방법을 알기쉽게 설명해줘

## Response:
생성 AI로 **템플릿화된 소설**을 만드는 가장 효과적인 방법은  
👉 \*\*“고정 구조(틀) + 가변 데이터 + 제어 프롬프트”\*\*를 분리하는 것입니다.  
아래를 순서대로 적용하면, 품질·재현성·확장성이 모두 좋아집니다.

* * *

1️⃣ 소설을 ‘글’이 아니라 **구조물**로 본다
----------------------------

먼저 소설을 자유로운 문장이 아니라 **조립 가능한 부품**으로 나눕니다.

### 기본 구조 예시

```
[세계관]
[주인공 설정]
[도입부]
[사건 발생]
[갈등 심화]
[클라이맥스]
[결말]
```

이 구조는 **절대 바꾸지 않는 고정 틀**입니다.  
AI는 이 틀 안에서만 글을 쓰게 됩니다.

* * *

2️⃣ 변하는 요소를 데이터로 분리한다 (핵심)
--------------------------

AI에게 “소설 써줘”라고 하면 매번 결과가 달라집니다.  
그래서 **변하는 요소를 전부 데이터화**합니다.

### 예시 (JSON 형태)

```json
{
  "genre": "미스터리",
  "tone": "음울하고 차분함",
  "setting": "비 오는 항구 도시",
  "protagonist": {
    "name": "윤서",
    "occupation": "기자",
    "flaw": "과거의 트라우마"
  },
  "theme": "진실은 항상 대가를 요구한다"
}
```

이렇게 하면:

*   같은 템플릿 + 다른 데이터 = 다른 소설
*   시리즈 제작, 대량 생성 가능

* * *

3️⃣ AI에게는 ‘작가’가 아니라 **엔진 역할**을 시킨다
----------------------------------

중요한 포인트입니다.

❌ 잘못된 방식

> “이 설정으로 소설 써줘”

✅ 효과적인 방식

> “아래 템플릿의 각 섹션을 채워라.  
> 톤, 분량, 시점, 금지사항을 반드시 지켜라.”

### 프롬프트 구조 예시

```
역할: 너는 장르 소설 생성 엔진이다.
목표: 아래 템플릿을 정확히 채운다.
규칙:
- 1인칭 시점
- 각 섹션은 300자 이내
- 감정 묘사는 행동으로만 표현
- 새로운 설정을 임의로 추가하지 말 것
```

👉 **창작 자유를 줄일수록 결과 품질은 올라갑니다.**

* * *

4️⃣ 템플릿은 ‘문장 틀’까지 내려간다
----------------------

고급 단계에서는 단락도 템플릿화합니다.

### 예시

```
[도입부]
- 장소 묘사 2문장
- 주인공의 현재 상태 1문장
- 불길한 징후 1문장
```

AI는 “어떻게 쓸까?”를 고민하지 않고  
\*\*“무엇을 채울까?”만 하게 됩니다.

* * *

5️⃣ 결과를 바로 쓰지 말고 ‘검수 단계’를 둔다
----------------------------

한 번에 완성하려 하지 말고, 2단계로 나눕니다.

1단계: 초안 생성  
2단계: 규칙 검사 및 수정

### 검수 프롬프트 예시

```
다음 텍스트가 아래 규칙을 위반했는지 점검하라.
- 시점 일관성
- 감정 직접 표현 여부
- 설정 추가 여부
위반 시 수정본만 출력하라.
```

이렇게 하면 **품질 편차가 급격히 줄어듭니다.**

## Prompt:
서스펜스로 만드는 방법을 구체적으로 순서대로 설명

## Response:
서스펜스 소설을 생성 AI로 만들 때는  
일반 소설보다 **정보 공개 타이밍과 긴장 구조**를 정밀하게 설계해야 합니다.  
핵심은 **“독자보다 주인공이 늦게 알게 하는 구조”** 또는  
\*\*“독자만 알고 주인공은 모르게 하는 구조”\*\*를 만드는 것입니다.

아래는 **생성 AI로 서스펜스 템플릿 소설을 만드는 실제 제작 순서**입니다.

* * *

생성 AI로 서스펜스 소설 만드는 7단계
======================

* * *

1단계: 핵심 긴장 요소 정의 (한 줄 공포)
-------------------------

먼저 소설의 긴장을 한 문장으로 정의합니다.

### 공식

```
[주인공]이 [위험한 상황]에 놓였는데
[더 큰 진실]을 모르고 있다.
```

### 예시

*   기억을 잃은 남자가 살인 용의자라는 사실을 모른다
*   구조 요청을 받은 소방관이 자기 집으로 출동한다
*   딸을 찾는 엄마가 범인의 집에서 일하고 있다

👉 이 한 줄이 **소설 전체의 긴장 엔진**이 됩니다.

* * *

2단계: 정보 비대칭 구조 설계 (서스펜스 핵심)
---------------------------

서스펜스는 **정보를 누가 알고 있느냐**로 만들어집니다.

| 구조 | 설명 | 효과 |
| --- | --- | --- |
| 독자 > 주인공 | 독자는 범인을 앎 | 긴장 지속 |
| 주인공 > 독자 | 주인공만 진실 앎 | 미스터리 |
| 둘 다 모름 | 단서 추적 | 추리형 구조 |

### 추천 구조 (AI 제작용)

```
초반: 독자만 진실 일부를 안다
중반: 주인공이 단서를 발견
후반: 진실이 뒤집힌다
```

* * *

3단계: 5구간 긴장 구조 만들기
------------------

서스펜스는 보통 **5단계 구조**가 가장 안정적입니다.

```
1. 평온
2. 이상 징후
3. 위험 증폭
4. 진실 접근
5. 폭로/전환
```

### 예시 구조

| 구간 | 내용 |
| --- | --- |
| 평온 | 평범한 일상 |
| 이상 | 수상한 문자 |
| 위험 | 누군가 따라옴 |
| 접근 | 단서 발견 |
| 폭로 | 가장 가까운 사람이 범인 |

* * *

4단계: 템플릿 구조 설계 (AI용)
--------------------

이제 AI가 채울 **고정 템플릿**을 만듭니다.

### 서스펜스 템플릿 예시

```
[1장: 평온]
일상과 주인공 소개

[2장: 이상 징후]
작은 불안 요소 등장

[3장: 위험 증가]
직접적인 위협 발생

[4장: 단서 발견]
주인공이 진실에 접근

[5장: 반전]
숨겨진 진실 공개
```

* * *

5단계: 변수 데이터(JSON) 생성
--------------------

이제 이야기의 재료를 데이터로 만듭니다.

```json
{
  "setting": "비 오는 해안 도시",
  "protagonist": {
    "name": "민준",
    "job": "택배 기사",
    "weakness": "야맹증"
  },
  "secret": "그가 배달하는 집은 모두 실종자와 관련되어 있다",
  "villain": "가장 친절한 고객",
  "theme": "일상 속의 공포"
}
```

* * *

6단계: 생성 프롬프트 설계
---------------

AI에게는 자유 창작이 아니라  
**구조 채우기 작업**을 시킵니다.

### 프롬프트 예시

```
역할:
너는 서스펜스 소설 생성 엔진이다.

목표:
아래 템플릿의 각 장을 작성한다.

규칙:
- 3인칭 제한 시점
- 각 장 400자 이내
- 직접적인 감정 표현 금지
- 행동과 환경으로 긴장 표현
- 새로운 설정 추가 금지
```

* * *

7단계: 긴장도 검수 단계 (매우 중요)
----------------------

AI가 만든 결과를 바로 쓰지 말고  
**긴장 체크 프롬프트**를 한 번 더 실행합니다.

### 검수 프롬프트

```
다음 텍스트에서
1. 긴장이 약한 부분
2. 정보가 너무 빨리 공개된 부분
3. 감정 직접 표현

을 찾아 수정하라.
수정된 텍스트만 출력하라.
```

가장 중요한 서스펜스 생성 공식 (실전 핵심)
=========================

### 공식 1: 시간 압박

```
위험 + 제한된 시간
```

예:

*   30분 안에 폭탄을 찾아야 한다

### 공식 2: 잘못된 믿음

```
주인공의 믿음 ≠ 진실
```

예:

*   친구를 믿었는데 친구가 범인

### 공식 3: 공간 제한

```
탈출 불가능한 장소 + 위험
```

예:

*   엘리베이터, 기차, 섬, 병원

| 감정 설명 과다 | 몰입도 감소 |
| 반전만 있고 과정 없음 | 허무함 |

2) 프롬프트 시스템 (복붙용 6종)
====================

아래 프롬프트는 **매번 동일하게 쓰고**, JSON만 바꾸면 자동 생성됩니다.


P1. 아웃라인 점검(Outline QA)
-----------------------

```

- 정보 공개 타이밍(암시→증거→확신)

{붙여넣기}
```

```
STORY_JSON:
{붙여넣기}
```

-------------------------------

```

STORY_JSON:
{붙여넣기}
SCENES_JSON:
{붙여넣기}
```

----------------

```

- 각 chapter는 STORY_JSON.meta.length_policy.chapter_char_limit 이내
- ‘보여주기’ 중심(소리/빛/촉감/거리)

출력 형식:
[1장: ...] ... (텍스트)
[2장: ...] ... (텍스트)
...
STORY_JSON:
{붙여넣기}
SUMMARY_JSON:
{붙여넣기}
```

---------------------------

```
아래 DRAFT를 검사하고 즉시 수정하라.
체크리스트:
1) 정보가 너무 빨리 공개되었는가?

3) 감정 직접 서술이 있는가?
4) 설정/연속성 위반이 있는가?
수정 후 최종본만 출력하라.
STORY_JSON:
{붙여넣기}
DRAFT:
{붙여넣기}
```
