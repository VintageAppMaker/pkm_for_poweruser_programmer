# 컴퓨팅 사고력과 바이브 코딩 정리

## 1. 컴퓨팅 사고방식 정의 표

| 개념 | 정의 | 쉬운 예제 |
|---|---|---|
| 분해(Decomposition) | 큰 문제를 더 작은 하위 문제로 나누어 다룬다. | “일정 앱”을 `날짜 선택 / 입력 / 목록 표시`로 쪼갠다. |
| 추상화(Abstraction) | 불필요한 세부를 제외하고 핵심 요소와 규칙만 남긴다. | 저장/로그인 같은 부가기능은 빼고 “입력→화면표시”만 먼저 만든다. |
| 패턴 인식(Pattern Recognition) | 반복되는 구조/규칙을 찾아 일반화한다. | 일정 항목은 항상 `날짜 + 내용` 구조로 반복된다고 정의한다. |
| 알고리즘(Algorithm) | 문제를 해결하기 위한 단계적 절차(순서)를 만든다. | `날짜 선택 → 내용 입력 → 추가 클릭 → 리스트 갱신` 순서를 정한다. |
| 자동화(Automation) | 반복 작업을 규칙에 따라 기계(프로그램)가 수행하게 한다. | “추가 버튼 클릭 시 리스트를 자동 갱신”하도록 만든다. |
| 디버깅(Debugging) | 오류의 원인을 찾고 수정해 정상 동작하게 한다. | “추가 눌러도 목록이 안 바뀜”을 재현하고 원인(렌더링/상태)을 고친다. |

---

## 2. 바이브 코딩에서 컴퓨팅 사고력을 사용하는 방법

### 2.1 바이브 코딩의 본질 정의
바이브 코딩은 다음과 같이 정의할 수 있다.

- 코드를 직접 설계하기보다  
- 하고 싶은 행동과 흐름을 자연어로 설명하면  
- AI가 그 설명을 바탕으로 코드를 생성하는 방식

이 방식에서는 문법 숙련도보다 **사고 구조의 명확성**이 결과의 품질을 좌우한다.

---

### 2.2 바이브 코딩에서 컴퓨팅 사고력이 중요한 이유
컴퓨터와 AI는 사람처럼 맥락을 추측하지 않는다.  
명확히 구조화된 사고를 전달해야 의도한 결과가 나온다.

컴퓨팅 사고력은 다음 역할을 한다.

- 요구사항을 기능 단위로 쪼갠다
- 구현 세부보다 목적을 우선 전달한다
- 작업의 순서를 명확히 한다
- 반복되는 구조를 하나의 규칙으로 묶는다
- 문제 상황을 언어로 정확히 진단한다

---

### 2.3 컴퓨팅 사고 요소별 바이브 코딩 활용 방식

| 컴퓨팅 사고 요소 | 바이브 코딩에서의 활용 방식 |
|---|---|
| 분해 | 기능을 버튼·입력·출력 단위로 나누어 설명 |
| 추상화 | 구현 방법은 생략하고 목적과 결과만 전달 |
| 패턴 인식 | 반복되는 데이터 구조나 동작을 하나로 정의 |
| 알고리즘 | 실행 순서를 말로 명확히 나열 |
| 디버깅 | 이상 동작을 결과 중심으로 설명 |

---

## 3. 실패하는 바이브 코딩 예시

### 예시 프롬프트
```
HTML이랑 JavaScript로 일정 관리 화면 만들어줘
```

### 결과 문제점
- 요구 범위가 지나치게 넓음
- 기능 우선순위가 없음
- 결과물이 과도하게 복잡해짐
- 수정 요청 시 전체 구조가 흔들림

### 실패 원인
- 문제 분해가 없음
- 목적과 범위가 불분명
- 컴퓨팅 사고 기반 설명 부재

---

## 4. 성공하는 바이브 코딩 구조 (컴퓨팅 사고 적용)

### 4.1 분해
```
필요한 기능은 3개다
1. 날짜 선택
2. 일정 내용 입력
3. 화면에 리스트로 표시
```

### 4.2 추상화
```
저장 기능은 제외한다
새로고침 시 데이터가 사라져도 괜찮다
```

### 4.3 알고리즘(순서)
```
날짜 선택 →
텍스트 입력 →
추가 버튼 클릭 →
아래 리스트에 즉시 표시
```

### 4.4 패턴 인식
```
일정 하나는
날짜 + 내용 구조로 반복된다
```

### 4.5 디버깅 설명
```
현재 문제는
추가 버튼을 눌러도
리스트가 갱신되지 않는다
```

---

## 5. 바이브 코딩에 최적화된 프롬프트 공식

아래 순서를 지켜 설명하면 결과 품질이 안정된다.

1. 만들고 싶은 것의 목적  
2. 필요한 기능의 개수 (최소화)  
3. 각 기능의 동작 순서  
4. 지금은 고려하지 않을 기능  
5. 현재 발생 중인 문제 (있다면)  

### 적용 예시
```
목적: 약속 정보 미리보기 화면
기능: 날짜/시간 입력, 장소 입력, 미리보기 출력
순서: 입력하면 즉시 미리보기에 반영
제외: 로그인, 데이터베이스 저장
문제: 지도 표시가 동작하지 않음
```

---

## 6. 왜 이 방식이 효과적인가
AI는 단순한 코드 생성기가 아니라 요구사항을 해석해 구현하는 역할에 가깝다.

- 구조가 명확하면 정확한 코드가 나온다
- 요구가 모호하면 과도하거나 엉뚱한 결과가 나온다

컴퓨팅 사고력은 AI에게 전달하는 **설계도이자 작업 지시서** 역할을 한다.

---

## 7. 핵심 요약
- 바이브 코딩에서 중요한 것은 문법이 아니다
- 핵심은 문제를 구조적으로 설명하는 능력이다
- 컴퓨팅 사고력은 AI에게 일을 정확히 시키는 기술이다
- 좋은 바이브 코딩은 좋은 기획 설명과 같다
