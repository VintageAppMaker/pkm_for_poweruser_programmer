웹앱 바이브 코딩 수칙

반드시 지켜야 할 프롬프트 문체

❝ 정답을 묻지 말고, 지금 만들 것을 지시하라 ❞

  • 틀린사례: “가장 좋은 방법으로 만들어줘”
  • 실질사례: “OO를 OO 방법으로 만들어줘”

바이브 코딩은 인간의 말로 전달하는 프로그래밍 명령이다. 대화가 아니다.

1. 수칙:

다음은 HTML + JavaScript 기반의 간단한 웹앱을 ‘바이브 코딩(vibe coding)’ 방식으로 만들 때 실무적으로 바로 사용가능한 행동 수칙(checklist)

1.1 파일 수칙 – “1개의 HTML 파일로 만들어라.”

<!doctype html>
<html>
<body>
  <h1>Vibe App</h1>
  <script>
    console.log('start');
  </script>
</body>
</html>

1.2 구조 수칙 – “이해 가능한 쉬운 구조로 만들어라”

<!-- HTML = 구조 -->
<div id="app"></div>

<script>
// state
let count = 0;

// logic
function increase() {
  count++;
  render();
}

// view
function render() {
  document.getElementById('app').innerHTML = `
    <p>${count}</p>
    <button onclick="increase()">+</button>
  `;
}

render();
</script>

1.3 CSS 수칙 – “OO색, OO구조, OO 테마로 CSS 적용해라”

body {
  font-family: system-ui;
  padding: 20px;
}

button {
  padding: 8px 12px;
}

1.4 데이터 수칙 – “모든 자료형태는 JSON으로 만들어줘”

const data = [
  { id: 1, title: 'idea' },
  { id: 2, title: 'vibe' }
];

1.5 저장수칙 - "localstorage로 저장해줘"

localStorage.setItem('memo', text);

1.5 버그추적 수칙 – “console.log로 과정을 출력해줘!”

console.log(state);
console.table(state.todos);
alert('여기까지 옴');

2. 프롬프트 구조

프롬프트 권장구조

[역할]
[목적]
[제약]
[출력 형식]
[행동 지침]

이 5개 블록만 있으면 바이브 코딩 프롬프트는 90% 완성이다.

역할 (Role)

AI의 성격을 제한한다 (중요)

너는 빠르게 프로토타입을 만드는 프론트엔드 보조 개발자다.

또는

너는 설계 설명 없이 바로 코드를 출력하는 HTML/JS 코딩 파트너다.

포인트

  • “설명가” X
  • “코딩 파트너” O

목적 (Goal)

지금 만들 ‘결과물’을 한 문장으로 고정

HTML과 JavaScript만 사용해
버튼을 누르면 숫자가 증가하는 간단한 웹앱을 만든다.

포인트

  • 기능 1~2개만
  • “예제”라는 말 쓰지 말 것 (완성물로 인식시킴)

제약 (Constraints)

바이브 코딩의 분위기를 강제하는 부분

- 단일 HTML 파일로 만든다
- 프레임워크 사용 금지
- 서버 없이 file:// 환경에서도 동작해야 한다
- ES6까지만 사용한다

이 블록이 없으면 → 설명 많아지고, 구조 과잉

출력 형식 (Output Format)

- 설명 없이 전체 HTML 코드만 출력한다
- 마크다운 설명 금지
- 코드블록 하나로 출력한다

또는

- [HTML 코드]
- [아래에 5줄 이내 사용법]

행동 지침 (Behavior Rules)

바이브의 맥락을 유지시키는 가이드

- 단순함을 최우선으로 한다
- 가독성이 좋으면 구조는 거칠어도 된다
- 돌아가기만 하면 성공으로 간주한다

3. 바이브 코딩 프롬프트 템플릿

기초 템플릿 예제

너는 빠르게 작동하는 프로토타입을 만드는 코딩 파트너다.

[목적]
HTML과 JavaScript만 사용해 ○○ 기능을 가진 간단한 웹앱을 만든다.

[제약]
- 단일 HTML 파일
- 외부 라이브러리 금지
- file:// 환경에서도 동작

[출력]
- 설명 없이 전체 HTML 코드만 출력

[지침]
- 단순함과 가독성을 우선
- 설계 설명은 하지 않는다

“좋은 프롬프트 vs 나쁜 프롬프트” 구조 비교

나쁜 구조

HTML과 JS로 좋은 예제를 만들어줘
최적화도 고려해줘

문제점

  • 목표 불명확
  • “좋은”이라는 추상어
  • 과설계 유도

좋은 구조

너는 빠르게 결과를 만드는 코딩 파트너다.

HTML과 JavaScript만 사용해
버튼 클릭 시 메시지를 화면에 출력하는 웹앱을 만든다.

- 단일 HTML 파일
- 설명 없이 코드만 출력

바이브 코딩 전용 키워드 사전 (자주 쓰면 효과 좋음)

프롬프트에 아래 단어를 의도적으로 섞으면 결과가 달라진다

키워드 효과
바로 동작하는 설계 최소화
단일 파일 구조 단순화
설명 없이 코드 집중
프로토타입 완성 강박 제거
거칠어도 된다 바이브 유지
빠르게 속도 우선

단계형 바이브 프롬프트 구조 (실전에서 많이 씀)

1단계: 바로 실행되는 최소 기능만 만든다
2단계: 같은 코드에서 기능 하나만 추가한다
3단계: 필요하면 정리한다

이렇게 쓰면 대화 자체가 바이브 코딩 세션이 된다.

바이브 코딩 프롬프트의 본질 (요약)

일반 프롬프트 바이브 코딩 프롬프트
“어떻게가 좋은가” “지금 뭘 만들까”
설명 중심 결과물 중심
정답 탐색 즉시 실행
설계 먼저 화면 먼저

한 문장으로 요약

바이브 코딩 프롬프트는
“설계 요청서”가 아니라
“지금 당장 만들 작업 지시서”다.

너는 빠르게 작동하는 프로토타입을 만드는 코딩 파트너다.

오늘 할 일 3개 체크 화면

너는 설계 설명 없이 바로 화면을 만드는 코딩 파트너다.

HTML과 JavaScript로
체크박스 3개가 있는 오늘 할 일 화면을 만든다.
체크하면 취소선이 그어진다.

제약:

텍스트 입력 → 화면에 카드 생성

아래에 카드 형태로 추가되는 화면을 만든다.

조건:
- 카드에는 입력한 텍스트만 표시

- 단일 HTML 파일

출력:
- 코드만 출력

URL 파라미터 → 화면 표시

너는 빠르게 결과를 보여주는 HTML/JS 파트너다.

file:///index.html?title=약속&msg=점심12시
형식의 URL 파라미터를 읽어
화면에 제목과 내용을 출력하는 페이지를 만든다.

제약:
- 서버 사용 금지
- 단일 HTML 파일

출력:
- 전체 HTML 코드만 출력

localStorage 메모장

날짜 선택 + 텍스트 표시 화면

HTML과 JavaScript로
날짜 선택 input과 텍스트 입력창이 있는 화면을 만든다.

버튼을 누르면
선택한 날짜와 내용을 아래에 정리해서 보여준다.

제약:
- 단일 HTML 파일
- 디자인은 기본만

출력:

클릭 시 색상 변경 박스

HTML과 JavaScript만 사용해
사각형 박스를 클릭할 때마다
색상이 순환해서 바뀌는 화면을 만든다.

조건:
- 색상은 3가지 이상
- 코드 단순하게 작성

출력:

숫자 입력 → 계산 결과 표시

HTML과 JavaScript로
숫자 두 개를 입력하면
합계를 바로 화면에 보여주는 간단한 계산기를 만든다.

제약:
- 버튼 하나만 사용
- 단일 HTML 파일

출력:
- 코드만 출력

프린트 전용 한 페이지 문서

HTML과 JavaScript로
제목과 내용을 입력하면
프린트용 한 페이지 문서 형태로 보여주는 화면을 만든다.

조건:
- 화면은 심플
- print 시 불필요한 버튼은 숨긴다

출력:
- 전체 HTML 코드만 출력

미리 정의된 문장 목록 중 하나를

제약:
- 문장은 JS 배열로 하드코딩
- 단일 HTML 파일

출력:
- 코드만 출력
